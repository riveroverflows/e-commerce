## Requirements Writing Rules (Design Docs Only)

Applies only when editing files under /docs or files named *requirements.md.

Hard constraints:
- DO NOT mention API, endpoints, HTTP, controllers, services.
- DO NOT mention DB, tables, schema, ERD, class names.
- Focus only on user scenarios and business rules.
- No implementation details.
- Must include explicit scope-freeze statements.
- Output language: Korean.
